HOW IT WORKS: Linear models

Why are the models linear?

Imagine that you have a lot of objects and you would like to assign some value to each object. For example, you have a set of bank card transactions, and you would like to understand which of these transactions were made by scammers. If you divide all operations into two classes and denote legitimate actions with zero and fraudulent actions with one, then you will have the simplest classification task. Imagine a different situation: you have exploration data that you would like to use to assess the prospects of different fields. In this case, based on a set of geological data, your model will, for example, estimate the potential annual profitability of a mine. This is an example of a regression task. The numbers that we want to match objects from our set are sometimes called targets.
Thus, classification and regression tasks can be formulated as a search for mapping from a set of objects to a set of possible targets. Obviously, just matching some objects to some numbers is a pretty pointless task. We want to quickly detect scammers or decide where to build a mine. So we need some kind of quality criterion. We would like to find a mapping that best approximates the true correspondence between objects and targets. What does "best" mean is a difficult question. We will return to it many times. However, there is a simpler question: among which mappings will we look for the best? There may be many possible mappings, but we can simplify the task for ourselves and agree that we want to look for a solution only in some predefined parameterized family of functions. This entire section will be devoted to the simplest such family — linear functions y = w1x1 + ... + wDxD + w0, where
y – target variable (target), (x1, ..., xD) – vector corresponding to the sample object (feature vector), (w1, ..., wD, w0) – model parameters. Signs are also called features. The vector is often called the vector of weights, since the prediction of the model can be viewed as a weighted sum of the features of an object, and the number w0 – free coefficient, or bias. A more compact linear model can be written as y = (x, w) + w0

Remark.

Let's figure out how such a model will work if D = 1. That is, our objects have exactly one numerical feature by which they differ. Now our linear model will look quite simple: y = x1w1 + w0. For the regression problem, we are now trying to approximate the value of y by some linear function of the variable x. And what will linearity mean for the classification problem? Let's remember the example of searching for fraudulent card transactions. Let's say we know exactly one numerical variable — the transaction volume. For binary classification of transactions into legitimate and potentially fraudulent, we will look for the so–called separating rule: where the value of a function is positive, we will predict one class, where it is negative, another. In our example, the simplest rule would be some kind of threshold value for the volume of transactions, after which it makes sense to mark the transaction as suspicious.
